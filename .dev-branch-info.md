# Rama Dev - Estructura Limpia

**Rama:** `dev`  
**Fecha creación:** 2025-11-29  
**Objetivo:** Mantener solo archivos esenciales para desarrollo

## Estructura en Raíz

### Directorios Esenciales (11)
- `app/` - Next.js App Router
- `components/` - Componentes React
- `config/` - Configuración activa
- `hooks/` - Custom hooks
- `lib/` - Utilidades y helpers
- `public/` - Assets estáticos
- `schemas/` - Schemas Zod
- `tests/` - Tests
- `types/` - Tipos TypeScript

### Directorio Workspace
- `_workspace/` - Utilidades, scripts, docs, datos

### Archivos de Configuración
- `package.json` - Dependencias
- `tsconfig.json` - Configuración TypeScript
- `next.config.mjs` - Configuración Next.js
- `tailwind.config.ts` - Configuración Tailwind
- `eslint.config.mjs` - Configuración ESLint
- `postcss.config.mjs` - Configuración PostCSS
- `jest.config.ts` - Configuración Jest
- `playwright.config.visitScheduler.ts` - Configuración Playwright
- `lighthouserc.json` - Configuración Lighthouse
- `README.md` - Documentación principal

## Archivos Ignorados

Los siguientes archivos/directorios están en `.gitignore`:
- `.temp-files/` - Archivos temporales
- `*.log`, `*.tsbuildinfo` - Archivos generados
- `static-build/` - Builds estáticos
- `_workspace/coverage/`, `_workspace/test-results/`, `_workspace/playwright-report/` - Reportes generados
- `.next/`, `node_modules/` - Builds y dependencias

## Uso

Esta rama está optimizada para desarrollo diario con una estructura limpia y organizada.
